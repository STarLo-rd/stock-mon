## üìù Documentation Standards

### File Naming Convention
- `UPPERCASE.md` - Root level docs (README, CHANGELOG)
- `PascalCase.md` - Documentation folder docs
- `lowercase-with-dashes.md` - Legacy or specific docs

### Markdown Standards
- Use GitHub-flavored Markdown
- Include table of contents for docs >500 lines
- Use code blocks with language specification
- Include examples for all concepts
- Keep line length readable (<120 chars preferred)

### Version Control
- Update CHANGELOG.md for all significant changes
- Include "Last Updated" date in documents
- Mark deprecated content clearly
- Keep old versions for reference
